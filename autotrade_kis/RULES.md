# Global Rules

## 1. Language

- All responses must be in **Korean (한국어)**.
- This includes code comments, commit messages, and documentation.

## 2. Coding Style

- Use **snake_case** for all variable and function names.
- Use **PascalCase** for all class names.
- Use **UPPER_SNAKE_CASE** for all constants.

## 3. Commit Messages

- All commit messages must be in **Korean**.
- Use the **Conventional Commits** specification.
- Format: `<type>: <subject>`
  - Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`

## 4. File Naming

- All file names must be in **snake_case**.
- Use **kebab-case** for directory names.

## 5. Documentation

- All documentation must be in **Korean**.
- Use **Markdown** format.
- Use **Google Style** docstrings for Python code.
